# Side Project 指南

我喜欢用代码建造工具，服务自己，帮助别人。我们这行，通常称之为 Side Project。
许多伟大的产品诞生于程序员的 Side Project. 比如 [GitHub](https://gist.github.com/defunkt/6443), [Unsplash](https://medium.com/who-what-why/how-side-projects-saved-our-startup-a83a80f3b3ae), [Instagram](https://www.molfar.io/blog/2020/side-projects), [Titter](https://www.molfar.io/blog/2020/side-projects)...
每次开启一个项目，虽然是 Side project, 但你内心其实隐隐在也在期待着，这个 Side Project 或许可以帮到除自己以外的人，
或许可以让这个世界变得更好，或许可能，这个产品价值足够大，甚至可以带来经济回报？然而现实是，99% 的 Side Project 都永远停留在自娱自乐的情况。经历热情满满，无人问津，最后弃坑这三个阶段。

如何提升 Side Project 的存活概率？你需要问自己下面四个问题：

[**如何判断一个点子的好坏？**](/idea)

[**如何快速把 Side Project 做出来？**](/build)

[**如何让我的 Side Project 帮助到更多人？**](/getting-users)

[**如何把 Side Project 变成能为自己带来收入的产品？**](/making-money)

如果对于它们没有一个很好的答案（特别是前三个）。这个 Side Project 最终通常会变成一次自娱自乐，最初的激情退却之后，不再维护，无法帮助到别人，甚至帮助不了自己。

这份小手册，记录下我对这几个问题的看法，以便复用。
