# Side Project 指南

我喜欢用代码建造工具，服务自己，帮助别人。我们这行，通常称之为 Side Project。做了许许多多的 Side Project 之后，我发现，以下四个问题总是重复出现。

**如何判断一个点子的好坏？**

**如何快速把 Side Project 做出来？**

**如何让我的 Side Project 帮助到更多人？**

**如何把 Side Project 变成能为自己带来收入的产品？**

如果对于它们没有一个很好的答案（特别是前三个）。通常这个 Side Project 最终通常会变成一次自娱自乐，最初的激情退却之后，不再维护，无法帮助到别人，甚至帮助不了自己。

这份小手册，记录下我对这几个问题的看法。我相信，这个 Side Project 会对我自己有所帮助，希望也对你有用。
