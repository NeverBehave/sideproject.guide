# Side Project 指南

或许你也知道，很多伟大的产品诞生于程序员的 Side Project. 比如 [GitHub](), [YC]()... 每次开启一个项目，虽然是 Side project, 但你内心其实隐隐在也在期待着，
这个 Side Project 或许可以帮到除自己以外的人，或许可以让这个世界变得更好，或许可能，这个产品价值足够大，甚至可以带来经济回报？
现实是，99% 的 Side Project 都永远停留在自娱自乐的情况。最近这些年来，我做了很多 SP，有几个“成功”了，大多数“失败“了。
渐渐的，我发现自己做的越来越少了，但是做成的概率大了一些，我想是有一些经验积累了。我想把这些总结一下，让自己有一些系统的关于如何做成 SP 的元知识。于是，我想写这样一份指南。

这份指南本身也是一个 Side Project, 我将总结一些这些年做 SP 以来自己所学到的，目的是希望帮助提高自己之后的 SP 成功率。如果可能，或许可以帮助到出像我一样喜欢做 SP 的人。



1. 点子是否值得被实现
2. 如何实现你的点子
3. 如何让更多人知道你的产品
4. 如何通过产品赚钱
